## 9.19 회의록

9.19 캡스톤 강의 후 회의사항

### 계획서 수정

9.19 캡스톤 계획서 발표 이후 계획서 일부 수정  
팀활동 - 프로젝트 계획서 작성하기 - 상세에서 점부파일로 확인 가능  
9.23일까지 수정 가능성 있음

-   추진 일정 상세화
-   계획 3단계 표현 수정 [서버 개발 -> 배포 준비]

### Git 분할

프론트엔드 / 백엔드 깃을 분할하여 관리하도록 의견 수렴됨

-   깃 프로젝트를 나눠서 관리
-   한 프로젝트에 폴더로 나누고, 브랜치로 관리
-   포크해서 배포용(메인) / 프론트 / 백엔드 프로젝트를 따로 관리하고, 풀리퀘(Pull Request)로 통합

### 정기 회의 일정

프로젝트 회의가 필요할 것으로 의견이 수렴되어 정기 회의 일정이 정해짐

-   금요일 캡스톤 강의 이후 1시간 정도
-   위치 미정

### 프로젝트 진행 논의

프로젝트 진행에 대한 건

-   백엔드 / 프론트 작업 분리
    -   공통
        -   프론트 - 백엔드 통신은 REST API로 진행하여, 두 작업 영역 분리
        -   작업할 때마다 총대 맬 사람 필요
    -   프론트
        -   웹 디자인 설계 급함 (대략적인 페이지 구성은 별도 작성)
    -   백엔드
        -   스프링부트 진행
